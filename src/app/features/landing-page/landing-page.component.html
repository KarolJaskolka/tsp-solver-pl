Travelling salesman problem

The travelling salesman problem (TSP) is an optimization task first formulated in 1930. TSP is about finding the shortest path between set of cities (or locations) that must be visited by salesman only once, apart from starting one. The goal is to reduce the distance traveled to maximium.

There are many algorithms and heuristics used to approximate the solution. Therefore I will present three chosen by myself methods - Nearest Neighbour, Local Search and Genetic Algorithm.

I have decided to prepare and use the dataset of the Polish voivodeships' capital cities (18) like eg. Warsaw (Masovian), Kraków (Lesser Poland), Poznań (Greater Poland) or Wrocław (Lower Silesian). The number of cities is small, because that will allow easier visualization of the program computing.
